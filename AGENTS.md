# Agent Guidelines

Scope: Entire repository.

- Follow the user's plain-UK-English, non-technical style and keep explanations simple.
- Maintain the required docs: README.md, ARCHITECTURE.md, CHANGELOG.md, SETUP.md, TROUBLESHOOTING.md, USAGE.md, PLAN.md, COSTS.md, ACCESSIBILITY.md, AGENTS.md, llms.txt, and any new ones you add.
- Keep the UI accessible (WCAG 2.2 AA) and modern; document accessibility in ACCESSIBILITY.md and keep tests runnable.
- No secrets in code or commits; use environment variables and .env.example when needed.
- Keep changes small and well-documented; update CHANGELOG.md and PLAN.md with each change.
- Prefer minimal dependencies; note costs/limits for any external service in COSTS.md.
